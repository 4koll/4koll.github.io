部署 Hexo

安装插件

创建仓库

  • 创建 blog 仓库用来存放 Hexo 项目 可私有
  • 创建 name.github.io 仓库用来存放静态博客页面, name 为用户名/组织名

生成密钥

deploy_key 为密钥名称, 可自定义

部署密钥

  • 复制 deploy_key 文件内容，在 blog 仓库 Settings -> Secrets -> New repository secret 页面上添加
    • 在 Name 输入框填写 HEXO_DEPLOY_PRI
    • 在 Value 输入框填写 deploy_key 文件内容
  • 复制 deploy_key.pub 文件内容，在 name.github.io 仓库 Settings -> Deploy keys -> Add deploy key 页面上添加
    • 在 Title 输入框填写 HEXO_DEPLOY_PUB
    • 在 Key 输入框填写 deploy_key.pub 文件内容
    • 勾选 Allow write access 选项

配置模板

  • Workflow 模版

    在 blog 仓库根目录下创建 .github/workflows/deploy.yml 文件

自定义域名

域名申请

  • Freenom 申请免费域名

    • 选择域名

    • 购买域名

  • 修改 Freenom 默认的 Nameservers 为 Cloudflare 的 DNS 服务器

域名解析

将域名托管在 Cloudflare 上, 通过配置 DNS 以及规则实现 https 访问

  • 配置 NDS

    • 添加 A 记录, 将自定义域指向name.github.io 对应的 IP 地址

    • 添加 CNAME 记录, 将自定义域指向 name.github.io

  • 创建规则

    以下 name.com 自行更换为根域名

    • http://name.com/ 始终使用 HTTPS 访问

    • http://www.name.com/ 重定向 https://name.com/

    • https://www.name.com/ 重定向 https://name.com/

创建 CNAME

在 blog/source 目录下创建 CNAME 文件, 文件内容只包含 name.com
